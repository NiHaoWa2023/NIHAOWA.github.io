一、翻转二叉树（226）

  • 给你一棵二叉树的根节点 root ，翻转这棵二叉树，并返回其根节点。

  • 思路：

    • 对于当前节点，交换其左右子树
    • 对其左右子树，又执行相同的操作

二、对称二叉树（101）

  • 给你一个二叉树的根节点 root ， 检查它是否轴对称。

  • 思路：
    • 比较root的左右子树是否对称
    • 如何比较左右子树是否对称：
      • 根结点相同（都为空或者值相等）
      • 左子树的左子树 和 右子树的右子树 对称，左子树的右子树 和 右子树的左子树 对称

三、二叉树的最大深度（104）

  • 给定一个二叉树 root ，返回其最大深度。

二叉树的 最大深度 是指从根节点到最远叶子节点的最长路径上的节点数。

  • 思路：
    • 二叉树最大深度 = max(左子树最大深度，右子树最大深度) + 1

四、二叉树的最小深度（111）

  • 给定一个二叉树，找出其最小深度。

最小深度是从根节点到最近叶子节点的最短路径上的节点数量。

说明：叶子节点是指没有子节点的节点。

  • 思路：
    • 二叉树最小深度 = min(左子树最小深度， 右子树最小深度) + 1
    • 但需要注意两种特殊情况：左子树为空 或者 右子树为空

五、平衡二叉树（110）

  • 给定一个二叉树，判断它是否是高度平衡的二叉树。

本题中，一棵高度平衡二叉树定义为：

一个二叉树每个节点 的左右两个子树的高度差的绝对值不超过 1 。

  • 思路：
    • ①自底向上：只要有一个节点不平衡，那么整棵树都不平衡
    • ②自顶向下：对于一棵平衡二叉树而言，其左子树和右子树都是平衡二叉树

六、二叉树的所有路径（257）

  • 给你一个二叉树的根节点 root ，按 任意顺序 ，返回所有从根节点到叶子节点的路径。

叶子节点 是指没有子节点的节点。

  • 思路：
    • 求从根节点到叶子节点的路径。可以分解成 求根节点的左右孩子到叶子节点的路径

七、路径总和（112）

  • 给你二叉树的根节点 root 和一个表示目标和的整数 targetSum 。判断该树中是否存在 根节点到叶子节点 的路径，这条路径上所有节点值相加等于目标和 targetSum 。如果存在，返回 true ；否则，返回 false

叶子节点 是指没有子节点的节点

  • 思路：
    • 要判断根节点的路径之和是否等于targetSum，只需判断其左子树或右子树的路径之和是否等于targetSum - root->val

八、从中序与后序遍历序列构造二叉树（106）

  • 给定两个整数数组 inorder 和 postorder ，其中 inorder 是二叉树的中序遍历， postorder 是同一棵树的后序遍历，请你构造并返回这颗 二叉树 。

  • 思路：

    • postorder数组中的最后一个元素，将inorder数组分成两部分，分别是左子树和右子树
    • 根据左子树和右子树的长度，可以将postorder数组分成 左子树和右子树 两部分
    • 对左子树和右子树重复相同操作

九、从前序与中序遍历序列构造二叉树（105）

  • 给定两个整数数组 preorder 和 inorder ，其中 preorder 是二叉树的先序遍历， inorder 是同一棵树的中序遍历，请构造二叉树并返回其根节点。
  • 思路：
    • 与第八题大致相同
    • 其优化在于，通过在原数组上修改下标区间，从而达到“划分左右子树的效果”，这样就不用创建新的vector了

十、最大二叉树（654）

  • 给定一个不重复的整数数组 nums 。 最大二叉树 可以用下面的算法从 nums 递归地构建:
  1. 创建一个根节点，其值为 nums 中的最大值。
  2. 递归地在最大值 左边 的 子数组前缀上 构建左子树。
  3. 递归地在最大值 右边 的 子数组后缀上 构建右子树。

返回 nums 构建的 最大二叉树 。

  • 思路：
    • 递归的思路 题目给的清晰明了
    • 划分左右子树，选择了与第九题同样的思路，一定要注意区间的定义

十一、合并二叉树（617）

  • 给你两棵二叉树： root1 和 root2 。

    想象一下，当你将其中一棵覆盖到另一棵之上时，两棵树上的一些节点将会重叠（而另一些不会）。你需要将这两棵树合并成一棵新二叉树。合并的规则是：如果两个节点重叠，那么将这两个节点的值相加作为合并后节点的新值；否则，不为 null 的节点将直接作为新二叉树的节点。

    返回合并后的二叉树。

    注意: 合并过程必须从两个树的根节点开始。

  • 思路：

    • 合并 root1 和root2 可以分解成 合并root1 和root2的根节点 + 合并 root1->left 和root2->left + 合并root1->right 和root2->right
    • 合并子树的方式和合并 root1 和root2方式一样，可以调用自己

十二、二叉搜索树（700）

  • 给定二叉搜索树（BST）的根节点 root 和一个整数值 val。

你需要在 BST 中找到节点值等于 val 的节点。 返回以该节点为根的子树。 如果节点不存在，则返回 null 。

  • 思路：
    • 递归思路：先搜索根节点，根节点不匹配就去左右子树搜索
    • 处理细节：搜索二叉树有个性质，if(root->val > val)，则val只能出现在左子树（当然也可能不存在）；反之，则val只能出现在右子树

十三、验证二叉搜索树（98）

  • 给你一个二叉树的根节点 root ，判断其是否是一个有效的二叉搜索树。

有效 二叉搜索树定义如下：

  • 节点的左子树只包含 小于 当前节点的数。
  • 节点的右子树只包含 大于 当前节点的数。
  • 所有左子树和右子树自身必须也是二叉搜索树。

  • 思路：

    • 思路一：二叉搜索树的中序遍历序列是递增的

    • 思路二：根据二叉搜索树的定义，只需判断左子树上所有的值都在[极小值，root->val）区间上，而右子树上所有的值都在(root->val,极大值] 区间上
